In this topic, we will discuss, PowerShell great way of dealing Cmdlets using Pipeline.
The Pipeline:
Allows multiple commands to be connected to each other, one-line sequence.
Represented in PowerShell by ‘|’
Use the output of one command to input for second command.
When two commands connected by pipeline, then first command (i.e. Cmd1) before pipeline is use for input for second command (i.e. Cmd2) after pipeline. Let’s see.



Here, Cmd1 generate the list of services that mentioned in C:\Services.txt. Now, the name of the services is placed to the pipeline. Here, PowerShell is deciding how to get these services name to Get-Service.
Here, the concept Pipeline parameter binding comes where Pipeline will search for a parameter of Get-Service that accepts any pipeline output. PowerShell has two methods it can use to get the output of Get-Content onto a parameter of Get-Service. The first method, the shell will try, is calledByValue; if that does not work, it’ll tryByPropertyName.

Pipeline input ByValue.

By this method, PowerShell first fetch the TypeName of first command (i.e. Cmd1) by piping Get-Member and then it will search parameter for second command (i.e. Cmd2) using –Full help(i.e. Help Get-Service –Full), to check if any parameter accepts that type of data from the pipeline ByValue.



So, we can see Get-Service does have parameter that accepts String from the pipeline ByValue. You can also try one more example as follows:
